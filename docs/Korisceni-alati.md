# Коришћени алати

## 1. Напомена

Овај документ садржи списак технологија, библиотека и развојних алата који су коришћени у пројекту **SideKick**, како у фази имплементације, тако и у претходним фазама пројекта.

Верзије су прикупљене из:

- конфигурације и зависности у репозиторијуму
- локалног развојног окружења у коме је пројекат покретан
- метаподатака сачуваних у пројектним артефактима

Када тачна верзија неког алата није могла поуздано да се реконструише само на основу репозиторијума, то је експлицитно назначено.

## 3. Додатно појашњење по групама

### 3.1 Backend технологије

Backend део система је реализован у језику **Python**, уз коришћење framework-а **Django**. Django је коришћен за:

- дефинисање модела базе података
- обраду HTTP захтева
- рендеровање HTML template-а
- реализацију аутентикације, рада са сесијама и CRUD операција

За покретање апликације коришћен је **Uvicorn** као ASGI сервер, док је за realtime комуникацију употребљена библиотека **python-socketio**.

### 3.2 База података и моделовање

У имплементационој фази коришћена је **SQLite** база података као локално складиште.  
У фази пројектовања базе података коришћен је **MySQL Workbench**, што је видљиво по артефакту `docs/SideKickDB.mwb`.

MySQL Workbench је коришћен за:

- цртање модела ентитета и веза
- пројектовање табела и атрибута
- дефинисање примарних и страних кључева
- припрему базе за каснију имплементацију у оквиру Django модела

### 3.3 Frontend технологије

Frontend део система реализован је без додатног JavaScript framework-а, већ уз коришћење:

- **HTML5** за структуру страница
- **CSS3** за визуелни изглед и responsive понашање
- **vanilla JavaScript-а** за динамичко понашање корисничког интерфејса

Исти технолошки стек је коришћен и за ранији HTML прототип документа `docs/sidekick-prototip.html`.

### 3.4 Realtime и екстензија

За функционалности које захтевају аутоматско освежавање приказа и сарадњу у реалном времену коришћена је комбинација:

- **python-socketio 5.8.0** на серверској страни
- **Socket.IO JavaScript Client 4.8.1** на клијентској страни

За Chrome екстензију коришћена је платформа **Chrome Extension Manifest V3**, уз API-је:

- **Side Panel API**
- **Storage API**
- **Tabs API**

Екстензија је дефинисана у директоријуму `extension/`, а њена верзија у `manifest.json` је **0.1.0**.

### 3.5 Развој, праћење измена и документација

За контролу верзија коришћен је **Git**.  
Документација је вођена у два облика:

- **Markdown** документи у оквиру репозиторијума
- **DOCX** документи за формалне спецификације и SSU материјале

За дељење прототипа и појединих докумената коришћен је и **PDF** формат.

## 4. Напомена о алатима чија тачна верзија није сачувана у репозиторијуму

На основу садржаја репозиторијума може се поуздано утврдити да је за моделовање базе коришћен **MySQL Workbench**, али тачна инсталирана верзија desktop апликације није експлицитно сачувана у репозиторијуму. Због тога је у табели наведено оно што се може поуздано извести: присуство `.mwb` модела и формат документа **1.4.4**.

Ако је за финалну предају неопходно навести и тачну desktop верзију тог алата, њу треба допунити на основу рачунара на ком је модел првобитно прављен.
